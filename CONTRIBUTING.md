# Contributing to Advanced GitHub API Client

We welcome contributions! Please follow these simple rules to ensure a smooth open-source collaboration:

1. **Clone & Setup**: Fork the repository, clone it locally, and run `flutter pub get` followed by `flutter pub run build_runner build --delete-conflicting-outputs` to generate all Freezed and Isar files.
2. **Code Cleanliness**: Before opening a Pull Request, always run `flutter format .` and `flutter analyze`. We maintain a strict zero-warning policy.
3. **Branching**: Create a clean branch for your work (e.g., `feature/awesome-new-view` or `bugfix/issue-12`).
4. **Pull Requests**: Keep your PRs focused on a single issue. Reference the issue number in your PR description.
